# RUNBOOK - run the mecca-pkg-llm pipeline end to end

## Feature: rich box recommendation (image and/or text)

The UI takes an **image, text, or both** and returns:
- a **rich product description** (item identification + container form + materials), from the
  Gemma vision/text step (`describe_image` / `describe_text`);
- a **packaging BOM** (PP/SP/TP) with components, materials, mass, **dimensions, shape
  (box / cylindrical / flat), and volume** - from the fine-tuned model;
- **sustainability metrics** - carbon (kg CO‚āāe), water (L), recyclability %, recycled
  content % - per tier and total.

**Accuracy rule (important):** carbon / water / recyclability are **NOT generated by the
model**. They are looked up deterministically from `catalog/env_catalog.json` (real
per-material intensities from MECCA's materials data, all 66 of our materials covered) and
scaled by the model's predicted mass - see `src/mecca_pkg_llm/enrich.py`. The model only predicts
*what material and how much*; the environmental numbers come from the catalog join.

data (`data/processed/phase3/` already has the richer target). Re-run steps 1‚Üí3 below.

## 0. One-time setup

```bash
cd mecca-pkg-llm
uv sync
cp .env.example .env      # fill AWS_PROFILE, AWS_REGION, BUCKET_NAME, SAGEMAKER_ROLE_ARN, HF_TOKEN
```
- `HF_TOKEN`: Gemma 3 is **gated** - accept the license on HuggingFace for
  `google/gemma-3-4b-it` AND `google/gemma-3-12b-it`, then create a read token.
- `SAGEMAKER_ROLE_ARN`: a role SageMaker can assume with S3 + ECR access (you can reuse the
  mecca-llm training role on account 471112736025).
- GPU quota: ensure `ml.g6.12xlarge` (or `ml.g5.12xlarge`) **training** + **endpoint** quotas
  are > 0 in your region. (Hopper/ml.p5 is NOT needed for a 12B QLoRA.)

## Which path do I need? (you usually do NOT retrain)

**Training is a one-time thing.** Once a model is trained it lives in S3 and you reuse it - you do
**not** re-run steps 1‚Äď4 every time you want to use the model.

- **Just want to use an already-trained model** (the normal case) ‚Üí skip to **step 5 (Serve / UI)**,
  or run `./redeploy.sh` to spin the last-trained model back up from `.env`. Then **delete the
  endpoint when done** (step 6). No data prep, no training.
- **Training the model for the first time, or you changed the data / target schema** ‚Üí run the full
  sequence **1 ‚Üí 6 once**, then you're back to the "just use it" path above.

Steps 1‚Äď4 below are the **one-time build**; steps 5‚Äď6 are the **everyday deploy ‚Üí use ‚Üí tear-down**
loop.

## 1. Data (free, local) - *one-time build*

```bash
uv run scripts/01_extract_chunk.py            # -> data/raw/chunk_10k.jsonl (10k products)
uv run scripts/02_preprocess.py               # -> products_clean.jsonl + materials_catalog.json (66 mats)
uv run scripts/03_format.py                   # -> phase3/{train,val,test}.jsonl (32k/1k/1k)
uv run eval/test_offline.py                   # sanity: metrics + train/infer prompt match
```

## 2. Train - cheap dev shakedown FIRST (~$1‚Äď2) - *one-time build*

```bash
uv run training/launcher.py --size dev --dry-run     # preview cost
uv run training/launcher.py --size dev               # 4B, 800 rows, ~$1.19 - proves the pipeline
```
Confirm in CloudWatch that: vision/projector freeze logged a non-zero count, LoRA modules
matched, loss decreased, model saved to S3. Note the model S3 URI in
`training/last_training_job.txt`.

## 3. Train - real run (~$50‚Äď70) - *one-time build*

```bash
uv run training/launcher.py --size real --epochs 2 --dry-run    # ~$66 for full 32k√ó2ep
# Cheaper option: fewer rows -> uv run training/launcher.py --size real --max-train-samples 16000
uv run training/launcher.py --size real --epochs 2
```

## 4. Evaluate + smoke test - *one-time build (after a fresh train)*

Deploy the endpoint (serverless = scale-to-zero, cheapest for eval):
```bash
uv run ui/endpoint.py deploy --name gemma3-bom-ep \
    --model-s3 s3://<bucket>/gemma_sft/models/<job>/output/model/ --serverless
```
```bash
uv run eval/evaluate.py --endpoint-name gemma3-bom-ep --limit 100   # two-condition metrics
uv run eval/smoke_test.py --endpoint-name gemma3-bom-ep --images ./sample_photos
```
Ship decision = the **caption** condition numbers in `eval/results/*/metrics.json`.

## 5. Interactive UI - *everyday use (no retraining)*

```bash
./redeploy.sh                       # spin the last-trained model back up from .env (no retrain)
uv run streamlit run ui/app.py
```
Image OR text input ‚Üí caption (step 1) + BOM (step 2). Deploy/DELETE the endpoint from the
sidebar.

## 6. ‚ö†ÔłŹ ALWAYS tear down - *everyday use*

```bash
uv run ui/endpoint.py delete --name gemma3-bom-ep
uv run ui/endpoint.py list        # confirm nothing is InService
```
A forgotten real-time `ml.g6.12xlarge` endpoint ‚Čą **$120+/day**. Serverless scales to zero,
but delete it anyway when finished.
